# 日程調整ページ（/yoyaku）セットアップ手順

リンクを共有した人だけが見られる非公開の日程調整ページです。
**すべて無料枠のサービスだけで動きます（追加課金なし）。**

| 役割 | サービス | 無料枠 |
|---|---|---|
| ページ・API のホスティング | Vercel（現行のまま） | Hobby プラン |
| 候補日・予約データの保存 | Upstash Redis（Vercel Marketplace） | 無料プラン（コマンド50万回/月） |
| 確認・リマインドメール | Resend | 月3,000通・日100通 |
| 前日リマインドの定期実行 | Vercel Cron | Hobby プランで1日1回まで無料 |

## 仕組み

**「イベント」単位で複数の予約ページを同時に運用できます。**
イベントごとに独立したタイトル・候補日・予約・共有URLを持ちます。

- **予約ページ**: `https://kabuexlabs.com/yoyaku/<イベントごとのトークン>/`
  - トークンはイベント作成時に自動発行。一致しないURLは 404 になるため、リンクを知らない人には存在ごと見えません。1つのURLが漏れても他のイベントには影響しません
  - 検索エンジンにも載りません（noindex ヘッダー付き・サイトマップ非掲載）
  - お客様は「ハンドルネーム・電話番号・メールアドレス・希望日程」を入力して予約
  - 予約された日程は即座に「予約済み」になり、他の方は選べなくなります（同時アクセスでも二重予約されません）
- **管理ページ**: `https://kabuexlabs.com/yoyaku/admin/`
  - 合言葉でログインし、イベントの作成・削除、候補日の追加・削除、予約の確認・キャンセルができます
  - ログイン状態はブラウザを閉じるまで保持されます（合言葉はURLに載らない安全な方式です）
- **メール**:
  - 予約時 → お客様に確認メール、`YOYAKU_ADMIN_EMAIL` に通知メール（どちらもイベント名入り）
  - 毎朝9時（日本時間）→ 全イベント横断で、翌日の予約があるお客様にリマインドメール、管理者に翌日の予定一覧

## 初回セットアップ（30分ほど）

### 1. データベース（Upstash Redis）をつなぐ

1. [Vercel ダッシュボード](https://vercel.com/dashboard) → このプロジェクト → **Storage** タブ
2. **Create Database** → **Upstash (Redis)** を選択 → **Free** プランで作成
3. プロジェクトに Connect すると環境変数（`KV_REST_API_URL` / `KV_REST_API_TOKEN`）が自動で追加されます

### 2. 環境変数を設定する

Vercel ダッシュボード → プロジェクト → **Settings → Environment Variables** で以下を追加します。

| 変数名 | 値 |
|---|---|
| `YOYAKU_ADMIN_KEY` | 管理ページの合言葉（十分長い推測されにくいもの） |
| `YOYAKU_ADMIN_EMAIL` | 予約通知を受け取るアドレス（未設定なら `info@kabuexlabs.com`） |
| `CRON_SECRET` | ランダム文字列。設定すると第三者がリマインドAPIを叩けなくなる（Vercel が自動で認証ヘッダーに付けます） |
| `BOOKING_PAGE_TOKEN` | （任意・旧構成からの引き継ぎ用）設定されていると、旧・単一ページ構成のデータを最初のイベントに取り込む際に共有URLとして再利用します。新規セットアップでは不要 |

### 3. メール送信（Resend）を設定する

1. https://resend.com で無料アカウント作成（クレジットカード不要）
2. **Domains** → `kabuexlabs.com` を追加し、表示される DNS レコード（TXT/MX）をドメインのDNS設定に追加して認証
3. **API Keys** で API キーを発行
4. Vercel の環境変数に追加:
   - `RESEND_API_KEY` = 発行したキー
   - `MAIL_FROM` = `ex Labs 予約窓口 <yoyaku@kabuexlabs.com>`（認証したドメインのアドレスなら何でも可）

> Resend 未設定でも予約機能自体は動きます（メールだけ送られません）。まず動作確認してからメールを設定してもOKです。

### 4. 再デプロイ

環境変数を追加したら **Deployments → 最新のデプロイ → Redeploy** で反映します。
前日リマインドの定期実行（毎日 日本時間 9:00）は `vercel.json` に設定済みなので、デプロイすれば自動で有効になります。

## 使い方

1. `https://kabuexlabs.com/yoyaku/admin/` を開き、合言葉でログイン（このURLをブックマークしておくと便利。合言葉はブラウザを閉じるまで再入力不要）
2. 「新しいイベントを作成」にイベント名を入れて作成（例: ウワサバナシ調査委員会 Case1 プレ公演 日程調整）。**共有URLが自動発行されます**
   - 「ご案内文」に集合場所・注意事項を書いておくと、**予約完了画面・確認メール・前日リマインドメール**に表示されます（予約前の画面には出ないので、場所は予約した人にだけ伝わります）
3. 「候補日を追加」で日付と時間帯（例: 14:00〜16:00）を登録。枠が多いときは「候補日を一括追加」に1行1枠（`2026-09-02 18:00〜` の形式）で貼り付け
4. ページ上部に表示される「お客様に共有するURL」をコピーして、お客様にメール等で送る
5. お客様が予約すると、確認メール（お客様宛）と通知メール（自分宛）が届く
6. 予約の前日 朝9時に、お客様へリマインドメールが自動送信される
7. 別のイベントを開きたくなったら「イベント一覧に戻る」→ 新規作成。イベントは何個でも並行運用できます

### よくある操作

- **予約をキャンセルしたい**: 各イベントの「予約一覧」→「キャンセル」。その枠は再び予約可能に戻ります
- **枠を一時的に締め切りたい**: 「候補日と空き状況」→「受付停止」。お客様の画面では「予約済み」と表示されます。「再開」でいつでも元に戻せます
- **候補日を消したい**: 「候補日と空き状況」→「削除」（予約が入っている枠は先にキャンセルが必要）
- **URLが漏れた気がする**: 該当イベントの「URLを再発行する」を押すと、古いURLは即座に無効になります
- **終わったイベントを片付けたい**: 「このイベントを削除する」（予約が残っている間は削除不可。先にキャンセルを）

## 無料枠の目安

- Upstash 無料枠は月50万コマンド。この用途（数十件/月の予約）なら1%も使いません
- Resend 無料枠は月3,000通。予約1件あたり最大4通（確認2＋リマインド2）なので月700件超まで余裕があります
- Vercel Cron は Hobby プランで「1日1回実行のジョブ2つまで」無料。ここでは1つだけ使用しています
